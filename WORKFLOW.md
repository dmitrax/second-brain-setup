
**`00-shared/SOUL.md`** — кто ты (2-3 абзаца):

## Часть 2б: Существующий проект (уже есть код и история)

Если проект уже на какой-то стадии — стандартный `/brain-init` создаст пустой vault,
и wiki будет расти только с новых сессий. Нужен один дополнительный шаг: **ретроспективный ingest** — "археологический" проход по существующей работе.

### Шаг 1. Стандартный init

```bash
cd ~/Projects/dotfiles   # существующий проект
claude
> /brain-init dotfiles
```

Отвечаешь на 6 вопросов как обычно. Статус проекта (вопрос 3) — честно: "в разработке, N месяцев, вот что сделано".

---

### Шаг 2. Скопируй ключевые файлы в raw/

В raw/ кладёшь то что **описывает проект** — не весь код (Claude Code видит код напрямую), а документацию, конфиги, описания архитектуры:

```bash
VAULT=~/Documents/second-brain-vault/dotfiles/raw

# README и документация
cp ~/Projects/dotfiles/README.md "$VAULT/"
cp ~/Projects/dotfiles/docs/* "$VAULT/" 2>/dev/null

# Конфиги (для dotfiles — это и есть продукт)
cp ~/.config/hypr/hyprland.conf "$VAULT/hyprland.conf"
cp ~/.config/waybar/config.jsonc "$VAULT/waybar-config.jsonc"

# Любые заметки или TODO если были
cp ~/Projects/dotfiles/NOTES.md "$VAULT/" 2>/dev/null
```

**Что класть в raw/:**
- README, docs/, CHANGELOG
- Конфиги которые уже работают (эталонные)
- Схемы БД, архитектурные диаграммы
- Старые заметки и TODO-файлы
- Транскрипты обсуждений если есть

**Что НЕ класть:**
- Весь исходный код (Claude видит его через файловую систему)
- node_modules, .git, build-артефакты
- Секреты и ключи

---

### Шаг 3. Ретроспективный ingest

```bash
> /brain-ingest raw/
```

Claude пройдёт по всем файлам и создаст начальную wiki. Для крупных проектов можно делать по файлу:

```bash
> /brain-ingest raw/README.md
> /brain-ingest raw/hyprland.conf
> /brain-ingest raw/waybar-config.jsonc
```

---

### Шаг 4. Попроси Claude провести "археологию" кодовой базы

После ingest сырых файлов — попроси Claude проанализировать сам проект:

```
> Проанализируй проект в ~/Projects/dotfiles/ и дополни wiki:
> 1. Какие архитектурные решения уже приняты (и почему — если понятно из кода)
> 2. Что точно работает и как
> 3. Что выглядит как незакрытый долг или временное решение
> 4. Структура проекта — ключевые файлы и их роль
> Создай заметки в dotfiles/wiki/ с ## For future Claude в каждой
```

Это аналог команды `/obsidian-architect` из системы Ghelbur — Claude читает
кодовую базу и пишет wiki с объяснениями "почему код такой".


### Итог: разница между новым и существующим проектом

| | Новый проект | Существующий проект |
|---|---|---|
| `/brain-init` | ✓ стандартно | ✓ стандартно |
| taskboard.md | создаётся пустым | создаётся пустым |

После этого workflow становится идентичным — ежедневная работа, `/brain-save`, git push.

## Когда обновлять CLAUDE.md

CLAUDE.md Блок 2 (правила проекта) — стабильный документ, меняется редко.
Обновляй его когда в проекте что-то фундаментально изменилось:

```
Изменился стек         → обнови раздел "Stack and tools"
Новое правило          → добавь в "Rules"
Новый запрет           → добавь в "Do not"
Новые команды проекта  → обнови примеры команд
```

Claude сам подскажет: если в ходе сессии ты говоришь "мы всегда делаем X"
или "никогда не делай Y" — Claude спросит: "Добавить это в CLAUDE.md?"

Обновление вручную:
```bash
# Просто открой файл и отредактируй Блок 2
nano ~/projects-code/[проект]/CLAUDE.md
# или попроси Claude:
> Обнови CLAUDE.md: добавь правило "backup before editing any config"
```

## Часть 3: Ежедневная работа

### Начало сессии

```bash
cd ~/Projects/dotfiles
git pull  # синхронизировать vault с другими устройствами
claude
```

Claude читает автоматически (~550 токенов):
- `CRITICAL_FACTS.md` — кто ты
- `dotfiles/_PROJECT.md` — что за проект + последняя сессия
- `dotfiles/taskboard.md` — приоритеты проекта

Ты сразу видишь:
```
Продолжаем dotfiles.
Последняя сессия (2026-06-01): настроили waybar.
Зависло 5 дней: bluetooth setup (причина: нет времени).
На неделе: bluetooth, затем аудио.
```

**Можешь сразу работать — никаких объяснений не нужно.**

---

### Во время сессии

**Обычная работа:**
Пишешь задачу естественным языком. Claude сам обратится к wiki когда нужен контекст.

```
> Хочу настроить bluetooth на Hyprland
→ Claude читает wiki/[заметки про bluetooth и audio] и работает
```

**Добавить новый источник в базу:**
```
> /brain-ingest raw/bluetooth-fix.md
```
Claude прочитает файл, обновит wiki, добавит [[wikilinks]], запишет в log.

**Добавить источник без команды (просто скинуть):**
```
> Вот конфиг который нашёл на GitHub: [вставляешь текст]
  Добавь в базу как референс
→ Claude создаст заметку в wiki/ и интегрирует
```

**Спросить что знаем по теме:**
```
> Что мы знаем про audio на Arch?
→ Claude ищет в wiki/, отвечает со ссылками на заметки
```

**Зафиксировать решение:**
```
> Мы решили использовать pipewire вместо pulseaudio — запиши решение
→ Claude создаст заметку в wiki/decisions/ с обоснованием
```

---

### Конец сессии

```bash
> /brain-save
```

Claude выполняет автоматически:
1. Создаёт лог сессии в `sessions/`
2. Обновляет (rewrite) затронутые wiki-заметки
3. Добавляет [[wikilinks]] между новыми и существующими заметками
4. Обновляет `dotfiles/taskboard.md` — что сделано, что осталось
5. Обновляет `_PROJECT.md` → блок "Последняя сессия"

После этого:
```bash
cd ~/Documents/second-brain-vault
git add . && git commit -m "$(date +%Y-%m-%d) dotfiles" && git push
```

## Часть 4: Работа с источниками (Ingest)

Любой материал который хочешь "дать Клоду" — сначала в `raw/`:

```
raw/
├── hyprland-dotfiles-ref.conf   ← конфиг с GitHub
├── bluetooth-arch-wiki.md       ← статья из Arch Wiki (Web Clipper)
├── созвон-с-командой.md         ← транскрипт через Whisper
└── моя-идея-про-workspace.txt   ← голосовая заметка → текст
```

Как попадает в raw/:
- **Конфиги и код** — копируешь вручную
- **Статьи** — Obsidian Web Clipper (расширение браузера → сохраняет как .md)
- **Аудио** — MacWhisper/Whisper → транскрипт → сохраняешь в raw/

Потом:
```bash
> /brain-ingest raw/bluetooth-arch-wiki.md
```

Что Claude делает с файлом:
- Извлекает ключевые знания
- **Перезаписывает** (не дублирует) существующие wiki-заметки
- Создаёт новые заметки если нужно
- Добавляет [[wikilinks]] между всеми затронутыми заметками

## Часть 5: Еженедельный аудит

Раз в неделю (например в пятницу):

```bash
cd ~/Projects/dotfiles
claude
> /brain-lint
```

Claude проверяет:
- Orphan-заметки (0 входящих ссылок) — предлагает где добавить [[ссылки]]
- Противоречия между заметками
- Устаревшие заметки (status: draft > 14 дней)
- Кросс-проектные связи → обновляет `connections.md`

После lint — открой **Obsidian Graph View** (`Ctrl+G`):

```
Что видеть:
● Кластеры — плотные облака заметок по проектам
● Хаб в центре — 00-shared/SOUL.md (к нему тянется всё)
● Мосты между кластерами — самые ценные кросс-проектные инсайты
● Изолированные ноды — orphans, нужны ссылки (уже нашёл /brain-lint)
```

График растёт — система умнеет.

## Часть 7: Obsidian — что смотреть

Obsidian — это только для просмотра. Vault пишет Claude, ты читаешь.

**Graph View (`Ctrl+G`):**
Открывай раз в неделю после `/brain-lint`. Видишь карту знаний.

**Dataview-запросы:**
Если установлен плагин Dataview, в любой заметке можешь написать:
```dataview
TABLE date, tags FROM "dotfiles/wiki"
WHERE contains(tags, "decision")
SORT date DESC
```
→ Таблица всех решений по проекту с датами.

**Поиск (`Ctrl+F` или `Ctrl+Shift+F`):**
Ищи по всему vault — по тегам, по содержимому, по названию.

**Плагины (обязательные):**

| Плагин | Для чего |
|---|---|
| Dataview | Таблицы по frontmatter |
| Templater | Шаблоны с автодатой |
| Web Clipper | Статьи → raw/ |

## Частые вопросы

**Нужно ли каждый раз объяснять Claude контекст?**
Нет. CLAUDE.md + vault делают это автоматически при старте.

**Claude не читает мои wiki-заметки при старте?**
Не все — только `CRITICAL_FACTS.md`, `_PROJECT.md`, `taskboard.md` (проектный). Остальные — по запросу когда нужны.

**Что если забыл сделать /brain-save?**
Знания потеряны из vault (но не из истории Claude Code если ты в той же сессии). Система не страхует от незакрытых сессий — это твоя ответственность.

**Как добавить скриншот или изображение?**
В Obsidian: Settings → Files → Attachment folder path → `raw/[проект]/assets/`. Вставляй картинки в заметки, Obsidian сохраняет их туда.

**Можно ли использовать с другим AI (Codex, Gemini CLI)?**
Да. Переименуй `CLAUDE.md` в `AGENTS.md` — работает во всех агентах поддерживающих этот стандарт. Vault не меняется.

**Vault разрастается — будет ли Claude читать всё подряд?**
Нет. Claude читает только то что нужно для конкретной задачи. `index.md` помогает навигировать не открывая лишнего.
