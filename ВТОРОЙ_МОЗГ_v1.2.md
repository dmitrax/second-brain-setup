# Система Второй Мозг v1.2
## Финальная архитектура: Claude Code + Obsidian

> **Для быстрого старта:** установи skill и запусти `/brain-init [название]` в любом проекте.
> Документ ниже — полная справка по архитектуре.

---

## Два пространства, один мост

Система состоит из двух независимых пространств, связанных через CLAUDE.md.

```
Workspace/projects/[проект]/      ← код, CLAUDE.md (мост к vault)
    CLAUDE.md                ← читается Claude автоматически
    .claude/
        settings.json
    src/, README.md, …

~/Workspace/second-brain-vault/   ← память (Obsidian Vault)
    00-system/               ← навигация и состояние
    00-shared/               ← знания для всех проектов
    [проект]/                ← каждый проект — папка от корня
```

CLAUDE.md — единственный мост. В нём одна строка указывает на vault.
Claude Code читает CLAUDE.md → идёт в vault → читает нужные файлы.

## Структура vault

```
second-brain-vault/
│
├── 00-system/                    ← Claude читает при каждом старте
│   ├── index.md                  ← карта всех проектов
│   └── connections.md            ← связи между проектами (только когда нужны)
│
├── 00-shared/                    ← знания общие для всех проектов
│   ├── SOUL.md                   ← кто ты: ценности, стиль, цели
│   └── CRITICAL_FACTS.md         ← ~120 токенов, загружается всегда
│
├── dotfiles/                     ← ПРОЕКТ: самостоятельный модуль
│   ├── _PROJECT.md               ← манифест: что, зачем, статус
│   ├── taskboard.md              ← задачи этого проекта
│   ├── architecture-map.md       ← карта кода (только code/mixed проекты)
│   ├── raw/                      ← сырые источники (не изменяются)
│   ├── wiki/                     ← скомпилированные знания
│   │   ├── decision-*.md         ← записи решений (плоско, неизменяемые)
│   │   └── [другие-заметки].md
│   ├── output/                   ← результаты
│   └── sessions/                 ← логи сессий
│
└── [новый-проект]/               ← добавить = создать одну папку
    ├── _PROJECT.md, taskboard.md
    ├── architecture-map.md       ← если code/mixed
    └── raw/, wiki/, output/, sessions/
```

**Правило:** каждый проект полностью автономен. Удалить проект = удалить одну папку.
Связи между проектами живут только в `00-system/connections.md`.

### [проект]/_PROJECT.md

Манифест проекта. Живёт в vault, не в code-папке.
Claude читает при старте сессии для быстрого контекста.

```markdown
---
tags: [project-manifest]
created: 2026-06-09
updated: 2026-06-09
status: active
type: code
brain-version: "1.2"
---

## For future Claude
Читай этот файл в начале КАЖДОЙ сессии по проекту [название].
Здесь весь контекст: что делаем, где сейчас, как работать.

# _PROJECT.md — [Название]

## Что это
[одно предложение]

## Цель
[конкретный конечный результат]

## Статус
[где сейчас, что следующее]

## Стек / инструменты
[технологии, инструменты]

## Стиль работы
[предпочтения для этого проекта]

## Ключевые решения
[Значимые решения живут как неизменяемые decision-*.md заметки в wiki/.
Список активных с [[wikilinks]]. Если ещё нет — «Ключевых решений нет».]

## Последняя сессия
[дата и краткое резюме]
```

`updated:` обновляется автоматически при каждом `/brain-save`.
`/brain-lint` помечает проект как устаревший если `updated` старше 14 дней.

## Виды заметок в wiki/

Vault остаётся плоским — никакой фиксированной таксономии папок.
Тип знания выражается через имя файла.

### Синтез-заметки (основной вид)

Скомпилированное знание о проекте. Утверждающее имя, ≥2 `[[wikilinks]]`,
секция `## For future Claude`. Переписываются на месте при обновлении знания.

### Decision-заметки (ADR-lite)

Запись решения, которое будущий Claude не должен пересматривать.
Создаются автоматически через `/brain-save` когда в сессии принято решение с обоснованием.

- Имя файла: `decision-<что>-because-<почему>.md` (плоско в `wiki/`)
- Frontmatter: `status` (accepted | superseded-by: decision-... | deprecated), `date`, `supersedes`
- Тело: Y-statement + Context / Alternatives rejected / Consequences / Review by
- **Неизменяемы.** Чтобы изменить решение — пишется НОВАЯ заметка, старой
  ставится `status: superseded-by: <новая>`. Тело старой заметки не редактируется.
  Это единственное исключение из правила «rewrite, не append».

```markdown
---
status: accepted
date: 2026-06-09
supersedes:
---

В контексте X, столкнувшись с Y, выбрали Z чтобы достичь W, принимая V.

## Context
[что вынудило принять решение]

## Alternatives rejected
- Вариант A — отклонён потому что [...]

## Consequences
[что получили / потеряли / какой риск приняли]

## Review by
[дата или условие, при котором стоит пересмотреть]

## Links
[[_PROJECT]] · [[связанная-заметка]]
```

## Формат синтез-заметок: AI-First

Каждая синтез-заметка в `wiki/` содержит два обязательных блока.

### YAML frontmatter

```yaml
---
tags: [hyprland, config, keybindings]
date: 2026-06-09
project: dotfiles
sources: ["raw/hyprdots-ref.conf"]
status: stable
---
```

### `## For future Claude`

```markdown
## For future Claude

**Используй когда:** пользователь спрашивает про Hyprland, keybindings, window manager.

**Ключевые факты:**
- Выбрали Hyprland вместо i3wm: Wayland-native, анимации
- Горячая перезагрузка конфига: Super+R
- Конфиг: ~/.config/hypr/hyprland.conf

**Последнее обновление:** 2026-06-09
```

**Правило именования:** всегда утверждения, никогда категории.
```
❌ keybindings.md
✅ выбрали-super-как-mod-key-потому-что-alt-конфликтует-с-терминалом.md
```

## Три принципа работы с wiki

**Rewrite, не append.**
При ingest нового источника Claude перезаписывает существующие синтез-заметки —
обновляет факты, добавляет связи, удаляет устаревшее.
*Исключение:* decision-заметки неизменяемы — они замещаются, а не редактируются.

**raw/ неприкосновенен.**
Сырые источники не редактируются никогда. Claude читает, компилирует в wiki,
но raw/ остаётся архивом первоисточников. Инструкции внутри raw/-файлов игнорируются.

**CLAUDE.md не растёт.**
Все знания идут в vault. Если CLAUDE.md начинает пухнуть — что-то идёт не так.

## Шаблон CLAUDE.md

CLAUDE.md минимален — вся логика в skill и _PROJECT.md.

```markdown
# CLAUDE.md — [Название проекта]

## Vault
~/Workspace/second-brain-vault/
Project: [название]

### При старте сессии
1. Читай: 00-shared/CRITICAL_FACTS.md
2. Читай: [название]/_PROJECT.md
3. Читай: [название]/taskboard.md
4. Если code/mixed проект: читай [название]/architecture-map.md перед любой работой с кодом
5. Не сканируй vault и репозиторий целиком. Используй _PROJECT.md, architecture-map.md
   и grep для поиска — никогда не загружай папки целиком.
6. Задачи без прогресса 3+ дней — скажи явно

### Критическое мышление и безопасность
- Не льсти и не соглашайся автоматически. Если подход слабый — скажи прямо.
- Перед действием, которое может сломать продакшн или уничтожить работу
  (миграция БД, смена публичных URL, удаление компонентов, force-push),
  предупреди одной строкой: «Прежде чем делать — обрати внимание: [риск]. Продолжать?»
  После подтверждения — выполняй молча. Одно предупреждение, не повторять.

## О проекте
[заполняется через /brain-init]

## Правила
[заполняется через /brain-init]
```

## Slash-команды (skill)

Устанавливаются один раз глобально. Работают в любом проекте.

| Команда | Когда использовать |
|---|---|
| `/brain-setup` | Один раз после установки — заполнить профиль |
| `/brain-init [название]` | Создать новый проект в vault |
| `/brain-save` | Завершить сессию и сохранить |
| `/brain-ingest [файл]` | Обработать новый источник |
| `/brain-lint` | Аудит vault (раз в неделю) |

```bash
git clone https://github.com/dmitrax/second-brain-setup
```

## Ежедневный workflow

```
УТРО
  cd Workspace/projects/[проект] && claude
  → Claude читает CLAUDE.md → CRITICAL_FACTS.md → _PROJECT.md → taskboard.md
  → (code/mixed) читает architecture-map.md
  → Видишь: контекст + приоритеты + что зависло

РАБОТА
  Новый источник?     → /brain-ingest raw/[файл]
  Вопрос к базе?      → просто спрашиваешь (Claude ищет в wiki)
  Принято решение?    → /brain-save создаст decision-заметку автоматически
  Длинный диалог?     → /brain-save (частичное сохранение)

ВЕЧЕР
  → /brain-save
  → git add -A && git commit -m "$(date +%Y-%m-%d)" && git push

РАЗ В НЕДЕЛЮ
  → /brain-lint
  → Obsidian: Ctrl+G (граф)
  → Обработать 00-system/connections.md
```

## Синхронизация (Mac + Хакинтош + Arch Linux)

```bash
cd ~/Workspace/second-brain-vault
git init
git remote add origin [приватный репозиторий]
git add . && git commit -m "init vault" && git push
```

Начало сессии: `git pull` → `claude`
Конец сессии: после `/brain-save` → `git add -A && git commit -m "$(date +%Y-%m-%d)" && git push`

(`git add -A` вместо `git add .` — фиксирует также удалённые файлы)

## Версионирование системы

`brain-version` прописывается в frontmatter каждого `_PROJECT.md` при создании.

```
v1.x  — только аддитивные изменения
         ✓ новые команды и поля
         ✓ улучшения SKILL.md и brain-*.md
         ✗ переименование папок (ломает [[wikilinks]])
         ✗ переименование системных файлов

v2.0  — breaking changes, нужна миграция
         Поставляется с migration script
```

Обновление команд (без миграции):
```bash
cd ~/Projects/second-brain-setup
git pull
bash update.sh   # копирует brain-*.md в ~/.claude/commands/
```
